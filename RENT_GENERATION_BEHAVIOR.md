# Rent Generation Behavior - Updated

## 📋 **Change Summary**

**Old Behavior:** Rent was automatically created when adding a new tenant  
**New Behavior:** Rent is ONLY created by the monthly cron job on the 1st of each month

---

## 🎯 **Why This Change?**

Previously, when you added a tenant, the system immediately created a rent payment record. This caused:
- Inconsistency (some rents created manually, some by cron)
- Confusion about when rent is due
- Difficult to predict rent generation

Now, **ALL rent generation happens in one place**: the monthly cron job.

---

## ⏰ **How Rent Generation Works Now**

### **Cron Job Schedule:**
- **Runs:** 1st of every month at 00:05 AM IST
- **Timezone:** Asia/Kolkata (Indian Standard Time)
- **Function:** `generateCurrentMonthRent()` in `backend/src/utils/scheduler.js`

### **What Happens on the 1st:**

The cron job processes **all current tenants** (where `is_current = 1`) and:

1. **Skips tenants who moved in THIS month**
   - First rent is always NEXT month
   - Example: Move in April 10 → First rent generated on May 1st

2. **Checks if rent already exists for current month**
   - Prevents duplicates
   - Uses SQL: `strftime('%Y-%m', due_date) = 'YYYY-MM'`

3. **Applies first rent logic if moved in PREVIOUS month:**
   - Moved in after 15th → Half rent (rounded)
   - Moved in on/before 15th → Full rent

4. **Creates rent record with:**
   - `due_date`: 10th of current month
   - `due_amount`: Calculated based on rules above
   - `paid_amount`: 0
   - `is_fully_paid`: 0

## 📅 **Rent Generation Examples**

### **Example 1: New Tenant (After 15th)**
```
Tenant moves in: March 20, 2026
House rent: ₹5,000/month

Timeline:
- March 20: Tenant added → NO rent created
- April 1 (cron): First rent created → April 10 due → ₹2,500 (half)
- May 1 (cron): Second rent created → May 10 due → ₹5,000 (full)
- June 1 (cron): Third rent created → June 10 due → ₹5,000 (full)
```

### **Example 2: New Tenant (On/Before 15th)**
```
Tenant moves in: March 10, 2026
House rent: ₹5,000/month

Timeline:
- March 10: Tenant added → NO rent created
- April 1 (cron): First rent created → April 10 due → ₹5,000 (full)
- May 1 (cron): Second rent created → May 10 due → ₹5,000 (full)
```

### **Example 3: Tenant Added on 1st of Month**
```
Tenant moves in: April 1, 2026
House rent: ₹5,000/month

Timeline:
- April 1: Tenant added → NO rent created
- April 1 (cron runs same day): Tenant moved in THIS month → SKIP
- May 1 (cron): First rent created → May 10 due → ₹5,000 (full)
  (Not half because move-in was on 1st, not after 15th)
```

### **Example 4: Tenant Added Late in Month**
```
Tenant moves in: April 25, 2026 (after 15th)
House rent: ₹5,000/month

Timeline:
- April 25: Tenant added → NO rent created
- May 1 (cron): First rent created → May 10 due → ₹2,500 (half)
- June 1 (cron): Second rent created → June 10 due → ₹5,000 (full)
```

## 🔧 **What Changed in the Code**

### **File: `backend/src/routes/tenants.js`**

**Removed (Lines 128-145):**

// OLD CODE - REMOVED
const moveInDate = new Date(move_in_date);
const dueDate = new Date(moveInDate.getFullYear(), moveInDate.getMonth() + 1, 10);
const now = new Date();
const currentMonthEnd = new Date(now.getFullYear(), now.getMonth() + 1, 0);

if (dueDate <= currentMonthEnd) {
  const firstRentAmount = moveInDate.getDate() > 15
    ? Math.round(house.rent_amount / 2)
    : house.rent_amount;

  db.prepare(`
    INSERT INTO rent_payments (tenant_id, house_id, due_date, due_amount)
    VALUES (?, ?, ?, ?)
  `).run(newTenant.id, house_id, dueDate.toISOString().split('T')[0], firstRentAmount);
}

## ⚠️ **Important Notes**

### **For New Tenants:**
1. When you add a tenant, **NO rent is created immediately**
2. Rent will appear **after the next cron run** (1st of following month)
3. If you need to manually create rent before the 1st, use the **"Generate Rent" button** on the Payments page

### **For Existing Tenants:**
- No change in behavior
- Rent continues to be generated on the 1st of each month

### **Manual Rent Generation:**
- You can still manually trigger rent generation using:
  - **Button:** Payments page → "Generate Rent" button
  - **API:** `POST /api/payments/generate-current-month`
- This is useful if:
  - Cron job fails
  - You need rent immediately
  - Testing purposes

---

## 🔄 **Backfill Missing Rent**

If you have tenants who moved in but don't have rent records yet, you can:

1. **Wait for next cron run** (1st of next month)
2. **Use Manual Generation Button** (Payments page)
3. **Use Backfill API** (generates all missing historical rent):
   ```
   POST /api/payments/backfill-missing
   ```

## 📊 **Business Logic Recap**

| Move-in Day | First Rent Month | First Rent Amount |
|-------------|------------------|-------------------|
| 1st - 15th | Next month | **Full rent** |
| 16th - 31st | Next month | **Half rent** (rounded) |

**Subsequent Rents:** Always full rent

**Due Date:** Always 10th of the month

**Generation Time:** 1st of each month at 00:05 AM IST

## 🆘 **Troubleshooting**

### **Problem: Tenant added but no rent showing**
**Solution:** This is expected! Rent will be created on 1st of next month. If you need it now, click "Generate Rent" button.

### **Problem: Cron job not running**
**Check:**
1. Server logs: `docker logs <container-name>` (Dokploy) or terminal output (local)
2. Look for: `[Scheduler] Running monthly rent record generation...`
3. Verify server timezone is IST

**Manual trigger:** Use "Generate Rent" button on Payments page

### **Problem: Wrong rent amount**
**Verify:**
1. Move-in date of tenant
2. House rent amount
3. First rent logic (after 15th = half)
